# Implit Roadmap - Phases 1-3

## Phase 1: Security Features

### 1.1 Malicious Package Detection
- Check packages against npm audit API
- Flag packages with known vulnerabilities
- Warn about packages with suspicious metadata (new packages, few downloads, unknown authors)

### 1.2 Vulnerability Scanning
- Integrate with `npm audit` 
- Check for CVEs in dependencies
- Show severity levels (critical, high, medium, low)

### 1.3 Supply Chain Attack Warnings
- Detect typosquatting (packages similar to popular ones)
- Flag packages created right after popular package removals
- Check maintainer reputation

## Phase 2: Workflow Integration

### 2.1 Git Pre-commit Hook
- Run implit on staged files before commit
- Block commits with hallucinated imports
- Auto-install: `npx @neurall.build/implit install-hook`

### 2.2 VS Code Extension
- Real-time import validation
- Inline warnings for bad imports
- Quick-fix suggestions
- Status bar indicator

### 2.3 GitHub Action
- Validate PRs automatically
- Comment on PR with issues found
- Block merge if hallucinations detected

## Phase 3: Multi-language Support

### 3.1 Python (pip)
- Parse Python imports (import X, from X import Y)
- Check against PyPI registry
- Validate local imports

### 3.2 Go (modules)
- Parse Go imports
- Check against proxy.golang.org
- Validate module paths

### 3.3 Rust (crates)
- Parse Cargo.toml dependencies
- Check against crates.io
- Detect unused or missing crates

## Implementation Priority

1. **GitHub Action** (highest impact, easiest to build)
2. **Python support** (large developer base)
3. **Malicious package detection** (security critical)
4. **Git pre-commit hook** (immediate value)
5. **VS Code extension** (best UX)
6. **Go/Rust support** (expanding ecosystem)
